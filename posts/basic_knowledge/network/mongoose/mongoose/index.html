目前使用

mongoose / Content-Type: multipart/form-data

提供的接口要求对参数进行RSA加密,Base64转码。而且请求头为multipart/form-data。

map.put(“appKey”,APP_KEY);

map.put(“request”,base64Str);

map.put(“sign”,sign);

Header authentication = new BasicHeader(“Authentication”, signRSA);

Header secutiryKey = new BasicHeader(“SecutiryKey”, privateKeyEncrypt);

Header company = new BasicHeader(“company”, companyStr);

方案一 ：

使用对称加密方式：

  1. 使用AES加密，增加秘钥下发接口，发送端生成AESKey 下发至盒子，盒子存储AESKey 后续通信使用该AESKey进行加密（支持更新，更新后使用新AESKey）
  2. 发送方请求时并对form key 和 value 进行加密 （除去图片文件及回复的查询信息外的其他字段是否需要加密待星际确认）
  3. 接收方获取 AESkey ，使用 AESkey 解密 form key 和 value
  4. 回复的查询信息使用对等的AESkey进行加密发送

方案二：

非对称加密+对称加密

生成公私钥对，公钥提供给星际，私钥在代码中

  1. 增加公钥获取接口，发送方获取公钥，盒子生成公私钥对，公钥发送给星际保存，私钥盒子保存，后续均使用该公私钥对。
  2. 发送方动态生成 AESKey 使用AESkey 对form key 和 value 进行加密（除去图片文件及回复的查询信息外的其他字段是否需要加密待星际确认） , 同时使用 base64 将密码转为秘钥字符串 AESKeyStr ,RSA 公钥进行加密 生成 AESKeySecert，将AESKeySecert放到http 请求头的SecutiryKey 中
  3. 接收方获取http请求头的 AESKeySecert ，使用 RSA秘钥进行解密操作获取 AESkey ，使用 AESkey 解密 form key 和 value
  4. 回复的查询信息使用对等的AESkey进行加密发送

其他：

  1. 图片等大文件的传输是否可以考虑生成图片的MD5，对MD5进行加密传输，接收接收文件计算MD5 并解密传输的MD5进行比对

  2. 后续请求不涉及文件传输时不在使用form data格式，使用json ，直接对发送的body 进行加解密

  3. 连接断了能否知道

  4. form data ，整体body data

B 方案： 和星际的通信方式有变化，主动推数据给到星际
